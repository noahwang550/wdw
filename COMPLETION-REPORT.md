### 2. Cron 定时任务配置

**位置**: `crons/daily-memory-update.yaml`

**配置详情**:
```yaml
schedule: "0 0 * * *"        # 每天 0:00 执行
timezone: "Asia/Shanghai"    # 亚洲/上海时区
command: ./update-daily.sh   # 执行更新脚本
enabled: true                 # 已启用
notify_on_failure: true       # 失败时通知群聊
notify_on_success: true       # 成功时通知群聊
timeout: 300                  # 5 分钟超时
retry:                        # 重试配置
  max_attempts: 2
  delay_seconds: 60
```

## 🎯 执行流程

### 正常流程（有记忆文件）
1. **0:00** - Cron 触发任务
2. **0:00:01** - 读取 `memory/YYYY-MM-DD.md`（前一天）
3. **0:00:02** - 检查文件存在且有内容
4. **0:00:03** - 执行 `generate-memories.py`
5. **0:00:05** - 生成 `memories.json`（包含所有记忆）
6. **0:00:06** - Git 检测变更
7. **0:00:07** - Git 提交（如有变更）
8. **0:00:10** - Git 推送到 GitHub
9. **0:00:15** - GitHub Pages 自动部署
10. **0:00:20** - 发送成功通知到群聊

### 边界情况（无记忆文件）
1. **0:00** - Cron 触发任务
2. **0:00:01** - 读取 `memory/YYYY-MM-DD.md`
3. **0:00:02** - 文件不存在
4. **0:00:03** - 记录警告日志
5. **0:00:04** - 任务正常退出（不报错）
6. **0:00:05** - 不发送通知（避免打扰）
